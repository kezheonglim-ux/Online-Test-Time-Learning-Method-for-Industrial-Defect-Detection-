## 1.2 Summary of Literature Review Directions

| Main Direction | Summary |
|---|---|
| Deep learning-based defect detection | Uses CNN, YOLO, ResNet, DETR, or transformer-based models to detect industrial defects such as PCB, wafer, photovoltaic, weld, and packaging defects. These methods improve detection accuracy but often `depend on labelled data` and `fixed training conditions`. |
| Incremental learning | Allows models to learn new defect categories over time while reducing catastrophic forgetting. However, most methods still `require staged training, labelled new samples, or controlled update sessions`. |
| Few-shot and unknown defect detection | Helps detect new or rare defect types when only 'limited samples are available'. The challenge is that most methods 'still need labelled few-shot samples or offline fine-tuning'. |
| Real-world anomaly detection | Highlights practical industrial problems such as lighting changes, unaligned images, subtle defects, and normal-reference comparison. These studies are highly relevant but `usually focus on benchmark/model design rather than full deployment workflow`. |
| Test-time learning / adaptation | Uses test-time information during inference to improve adaptation. This is close to this project direction, but many existing studies `focus on segmentation or classification rather than a complete online defect detection workflow`. |
| Low-code implementation | This is the `weakest area` in existing research. Most studies focus on algorithm performance and `do not explain how the model can be deployed in a user-friendly low-code industrial workflow`. |


## 1.3 Modelling Option Analysis

| Option | Type | Strength | Weakness | Relevant Reference |
|---|---|---|---|---|
| ResNet / WideResNet | CNN feature extractor | Commonly used as a stable pre-trained feature extractor in PatchCore-style industrial anomaly detection. Suitable for memory-bank feature embedding. | `Not designed as a detector` by itself. Detection/localization requires additional anomaly scoring, nearest-neighbor search, or classifier logic. | Roth et al. (2022), Towards Total Recall in Industrial Anomaly Detection. https://arxiv.org/abs/2106.08265 <br> Anomalib PatchCore Documentation. https://anomalib.readthedocs.io/en/v2.0.0/markdown/guides/reference/models/image/patchcore.html |
| EfficientNet | CNN feature extractor | Good accuracy-efficiency balance. Its compound scaling balances depth, width, and image resolution, making it useful when model size and inference efficiency matter. | Feature extraction quality depends on selected layer and transferability to defect images. Less directly connected to memory-bank anomaly detection than WideResNet/PatchCore. | Tan & Le (2019), EfficientNet: Rethinking Model Scaling for Convolutional Neural Networks. https://arxiv.org/abs/1905.11946 |
| Vision Transformer / ViT | Transformer feature extractor | Strong global feature representation and can perform well when pre-trained on large datasets. Useful for capturing long-range image relationships. | Usually heavier than CNN backbones and may `require more compute and careful tuning`. For small industrial datasets, `direct training may be difficult`. | Dosovitskiy et al. (2020), An Image is Worth 16x16 Words: Transformers for Image Recognition at Scale. https://arxiv.org/abs/2010.11929 |
| Faster R-CNN | Two-stage object detector | Strong localization accuracy and mature object detection framework. Region Proposal Network helps generate high-quality object proposals. | `Slower and more complex` than one-stage detectors such as YOLO and SSD. Requires labelled bounding boxes if used for supervised defect detection. | Ren et al. (2015), Faster R-CNN: Towards Real-Time Object Detection with Region Proposal Networks. https://arxiv.org/abs/1506.01497 |
| SSD | One-stage object detector | Lightweight and fast. Uses one network pass to predict object classes and bounding boxes, making it suitable for real-time detection. | Usually `less accurate than stronger modern detectors for small or subtle defects`. Requires labelled bounding boxes if used as a supervised detector. | Liu et al. (2016), SSD: Single Shot MultiBox Detector. https://arxiv.org/abs/1512.02325 |
| DETR / RT-DETR | Transformer-based object detector | DETR simplifies object detection as an end-to-end set prediction task. RT-DETR improves real-time transformer-based detection performance. | `More complex` and may need `more compute and tuning`. Still requires labelled detection data when used as a supervised detector. | Carion et al. (2020), End-to-End Object Detection with Transformers. https://arxiv.org/abs/2005.12872 <br> Zhao et al. (2024), DETRs Beat YOLOs on Real-time Object Detection. https://arxiv.org/abs/2304.08069 |
| CLIP / DINOv2 feature extractor | Foundation-model embedding | Strong general representation. Useful for zero-shot, few-shot, or general feature extraction settings. DINOv2 provides robust self-supervised visual features, while CLIP supports image-text transferable representations. | `Larger and heavier for deployment`. Feature design and inference cost may be more complex than YOLO/CNN backbones. May be over-complex for a low-code real-time workflow. | Radford et al. (2021), Learning Transferable Visual Models From Natural Language Supervision. https://arxiv.org/abs/2103.00020 <br> Oquab et al. (2023), DINOv2: Learning Robust Visual Features without Supervision. https://arxiv.org/abs/2304.07193 |
| YOLO26 | Recent YOLO-family real-time detector / feature extractor | Fast and deployment-friendly. Suitable for real-time visual workflow and can be frozen for feature extraction in this project. | Specific academic references for `YOLO26` may be limited. If used as a supervised detector, it `requires labelled bounding boxes`. | Mao et al. (2025), YOLO Object Detection for Real-Time Fabric Defect Inspection in the Textile Industry: A Review of YOLOv1 to YOLOv11. https://www.mdpi.com/1424-8220/25/7/2270 |

Based on the comparison, `YOLO26 was selected` in this project because the system requires a `practical` and `efficient visual feature extractor` that can be integrated with Flask API and CiRA CORE. Compared with heavier two-stage detectors such as Faster R-CNN and transformer-based detectors such as DETR, YOLO-family models are more suitable for real-time visual inspection workflows. In this project, `YOLO26 is used as a frozen feature extractor rather than a fully retrained supervised detector`. This `reduces training cost` while allowing the extracted image features to be compared with the normal memory bank for anomaly detection.


## 1.4 Comparison of Alternative Learning Methods

| Method | How It Works | Strength | Weakness | Challenge Implemented to YOLO26 | Relevant Reference |
|---|---|---|---|---|---|
| Offline retraining | Collect new data, label it, and retrain or fine-tune the model. | Stable and controlled because the model is updated in a planned training stage. | Slow; requires labelled data; `not suitable for fast deployment changes`. | High. If YOLO26 is retrained as a detector, labelled defect images and bounding-box annotations are required. This is time-consuming for industrial defect cases. | Plested & Gedeon (2022), Deep Transfer Learning for Image Classification: A Survey. https://arxiv.org/abs/2205.09904 |
| Transfer learning / fine-tuning | Fine-tune a pre-trained model on a target defect dataset. | Strong when enough labelled examples are available. | Needs labelled defect data and `may overfit when the dataset is small`. | Medium to High. YOLO26 can be fine-tuned, but the process requires labelled samples, training configuration, validation, and risk control. | Plested & Gedeon (2022), Deep Transfer Learning for Image Classification: A Survey. https://arxiv.org/abs/2205.09904 |
| Incremental learning | Learn new classes or new data stages while reducing forgetting of old knowledge. | Useful for adding known new defect classes over time. | Usually `requires staged training, labelled new samples, and forgetting-control methods`. | High. YOLO26 would need a structured update strategy, such as distillation, replay, or feature alignment, which increases system complexity. | Sun et al. (2024), An Incremental Knowledge Learning Framework for Continuous Defect Detection. IEEE TIM. <br> Wang et al. (2022), Online Detection of Weld Surface Defects Based on Improved Incremental Learning Approach. https://doi.org/10.1016/j.eswa.2021.116407 |
| Few-shot learning | Learn new defect categories from a few labelled examples. | Useful when only a few labelled defect samples exist. | Still needs `labelled support` samples and a `carefully designed learning strategy`. | High. Few-shot learning with YOLO26 would need a few-shot training setup and labelled support data for each new defect class. | Zhu et al. (2024), Few-Shot Class Incremental Learning for Packaging Defects with Double Class Feature Interaction. https://ieeexplore.ieee.org/document/10587516 |
| Unsupervised anomaly detection | Learn normal features and detect deviations from normal patterns. | Good when labelled defect samples are limited or unavailable. | Fixed memory bank and threshold `may not adapt well when deployment conditions change`. | Low to Medium. This is more suitable for YOLO26 because YOLO26 can be frozen as a feature extractor, and the extracted features can be compared with a normal memory bank. | Roth et al. (2022), Towards Total Recall in Industrial Anomaly Detection. https://arxiv.org/abs/2106.08265 |
| Self-supervised learning | Learn image representations from pretext tasks without labels. | Can improve feature quality without manual labels. | Requires `additional training design, validation, and careful selection of pretext task`. | High. It would require training or adapting the representation model before deployment, which is more complex than using YOLO26 directly as a frozen feature extractor. | Oquab et al. (2023), DINOv2: Learning Robust Visual Features without Supervision. https://arxiv.org/abs/2304.07193 |
| Active learning | The model selects uncertain samples and asks humans to label them. | Reduces labelling effort by focusing on useful samples. | Needs `human-in-the-loop labelling` and a `continuous feedback process`. | Medium to High. It can be added later, but it requires user confirmation, sample storage, annotation workflow, and periodic model update. | Settles (2009), Active Learning Literature Survey. https://minds.wisconsin.edu/handle/1793/60660 <br> Ren et al. (2020), A Survey of Deep Active Learning. https://arxiv.org/abs/2009.00236 |
| Rule-based inspection | Use fixed thresholds, edge, colour, texture, or shape rules. | Simple, fast, and explainable. | `Weak for complex defects`, changing lighting, and product variation. | Low. Easy to implement, but it does not fully use YOLO26 feature representation and may not handle complex industrial defects well. | Traditional image-processing methods are commonly used as baseline methods, but they are less suitable for adaptive defect detection compared with deep feature-based methods. |
| Online test-time learning with memory bank | Use YOLO26 as a frozen feature extractor, compare incoming image features with normal memory bank, and update trusted normal-like features during deployment. | More suitable for changing deployment conditions without full retraining. | `Risk of memory contamination` if abnormal samples are wrongly added as normal. | Medium. Easier than full retraining or incremental learning because only the adapter, threshold, or memory bank is updated while YOLO26 remains frozen. | Wang et al. (2020), Tent: Fully Test-Time Adaptation by Entropy Minimization. https://arxiv.org/abs/2006.10726 <br> Wang et al. (2022), CoTTA: Continual Test-Time Domain Adaptation. https://arxiv.org/abs/2203.13591 |

Based on this comparison, online test-time learning with a memory bank is selected because it is more practical for this project. Since YOLO26 is already used as the feature extractor, the system `does not need to retrain the full model during deployment`. Instead, YOLO26 can remain frozen, while the normal memory bank, threshold, or lightweight adapter is `updated using trusted normal-like samples`. This `reduces the implementation difficulty` compared with offline retraining, transfer learning, incremental learning, and few-shot learning, which usually require labelled defect samples and separate training stages.

However, the main challenge of the selected method is update reliability. If abnormal images are wrongly treated as normal and added into the memory bank, the memory bank may become contaminated. Therefore, the update should only be performed when the image is confidently classified as normal or confirmed as normal by the user.

Based on the comparison, `CiRA CORE was selected` because this project `requires a visual low-code workflow that can connect with the Flask API, trigger image inspection, display the current image, and show the anomaly result, score, threshold, and category`. Node-RED is a strong alternative for IoT/API workflow, while KNIME and Orange are more suitable for data science and offline ML analysis. Streamlit and Gradio still requires coding and are less suitable for node-based industrial workflow control. Therefore, CiRA CORE is the most suitable option for this project because it is already integrated with the working inspection flow and matches the low-code deployment objective.
