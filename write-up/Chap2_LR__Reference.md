# Chapter 2: Literature Review
## 2.1 Introduction
This chapter reviews previous studies related to industrial defect detection, incremental learning, few-shot and unknown defect detection, real-world anomaly detection, test-time learning and low-code implementation. These areas are relevant to this research because the proposed study focuses on developing an online test-time learning method for industrial defect detection using a low-code platform.
The review first discusses the development of deep learning methods for industrial defect detection. It then examines incremental learning approaches that allow models to learn new defect categories while reducing catastrophic forgetting. Next, studies on few-shot learning, unknown defect detection, real-world anomaly detection, and test-time learning are reviewed. Finally, the chapter identifies the research gaps and explains how this study is positioned within the existing literature.

## 2.2 Deep Learning for Industrial Defect Detection
Deep learning has been widely used in industrial defect detection because it can automatically learn useful visual features from images. Compared with traditional image-processing methods, deep learning reduces the need for manual feature design and can handle more complex defect patterns. Convolutional neural networks and object detection models have been applied in many inspection tasks, including PCB defect detection, wafer defect classification, photovoltaic cell defect detection and weld surface defect recognition.
In PCB inspection, recent studies have improved defect detection by using optimized object detection models, feature enhancement modules, multi-scale feature fusion and domain adaptation strategies. These methods show that deep learning can support both accuracy and speed requirements in industrial inspection especially when detecting small or visually weak defects (Ge et al., 2025; Yan et al., 2025). However, PCB production may involve changing product batches, different manufacturing conditions and newly appearing defect categories. This means that a model trained only under a fixed condition may not remain reliable during continuous production.
In wafer inspection, deep learning has also been used to classify wafer map defect patterns and identify unknown wafer defects. Early incremental wafer studies used ResNet-based models to support continuous learning from streaming data while later studies explored unknown defect detection, few-shot contrastive learning and memory-based updating methods (Ni et al., 2026; Yu et al., 2021; Yue et al., 2026; Zhao et al., 2024). These works show that wafer inspection requires more than high initial accuracy. The model also needs the ability to respond when new or rare defect patterns appear during production.
Other industrial applications show similar needs. In photovoltaic cell inspection, class-incremental learning has been used to update defect detection models when new defect categories appear in a dynamic data stream (Wang, Chen, et al., 2024). In weld surface inspection, online defect detection has been studied to improve real-time processing and reduce the delay caused by complex feature extraction and classification (Wang, Zhang, et al., 2022). These studies show that industrial defect detection is moving from static inspection toward more adaptive and real-time systems.
Although deep learning methods have achieved strong performance, most of them still depend on fixed datasets and offline training. Once the model is deployed, it usually does not update unless retraining or fine-tuning is carried out. This creates a practical limitation because production conditions may change and new defect types may appear after deployment.

## 2.3 Incremental Learning in Defect Detection
Incremental learning has been introduced to reduce the limitations of static learning. Instead of training a model only once, incremental learning allows the model to learn new defect categories or new data distributions while retaining previously learned knowledge. This is useful in industrial inspection because defect types may increase gradually as production continues.
A major challenge in incremental learning is catastrophic forgetting. This happens when a model learns new defect categories but loses its ability to recognize old categories. To address this issue, several strategies have been used including knowledge distillation, memory replay, feature alignment, teacher-student learning and feature decoupling (Shen et al., 2025; Sun et al., 2024; Wang, Chen, et al., 2024; Xiong et al., 2024; Zhao et al., 2024).
Knowledge distillation is commonly used to help the updated model retain old knowledge while learning from new data. In continuous defect detection, feature-level and logit-level alignment have been used to reduce forgetting when few-shot defect categories are introduced (Sun et al., 2024). Elastic distillation and heterogeneous distillation have also been explored to balance the learning of new defect classes with the retention of old class knowledge (Shen et al., 2025). These methods show that distillation can support model stability during incremental updates.
Other studies focus on balancing old and new class knowledge more effectively. Adaptive dual-teacher learning has been used to adjust the importance of old and new defect classes during incremental learning while feature decoupling methods have been used to reduce inaccurate knowledge transfer from old-class models (Xiong et al., 2024). In photovoltaic cell defect detection, multi-scale feature decoupling and similarity distillation have been used to control how important defect regions and background regions are handled during model updating (Wang, Chen, et al., 2024).
Memory-based approaches have also been explored. Reference buffers and memory replay mechanisms can preserve representative information from previous learning stages helping the model reduce forgetting when new defects are introduced (Yue et al., 2026; Zhao et al., 2024). These methods are useful because they reduce the need to retrain the model from the beginning whenever new data appear.
Overall, incremental learning improves the adaptability of defect detection models. However, most existing methods still update the model through staged learning sessions. In many cases, new data must be collected, labelled, and used in a separate training stage. This means that incremental learning is more flexible than static learning but it may still not fully support direct adaptation during inference.

## 2.4 Few-Shot and Unknown Defect Detection
In real industrial production, new defect types may appear with only a small number of samples. This makes few-shot learning and unknown defect detection important. Few-shot learning allows the model to learn from limited labelled data while unknown defect detection focuses on identifying defect types that were not included in the original training set.
Few-shot class-incremental learning has been applied to packaging defect detection where new defect categories are learned from limited samples while the model tries to reduce forgetting of old categories (Zhu et al., 2024). Transformer-based detectors have also been introduced into incremental few-shot surface defect detection to improve the learning of new categories and reduce knowledge ambiguity between old and new classes (Yang et al., 2025).
In wafer inspection, unknown defect detection has received increasing attention because semiconductor manufacturing processes are complex and new defect patterns may appear unexpectedly. Incremental learning, contrastive learning, and few-shot learning have been combined to improve the detection of unknown wafer defects when only limited samples are available (Ni et al., 2026; Zhao et al., 2024). These methods are relevant to this research because they show the need for models that can handle new or unfamiliar defect patterns without full retraining.
Incremental meta-learning has also been explored for PCB defect detection especially for tiny defects and new defect types. The combination of meta-learning and incremental learning can help the model adapt faster when only limited new data are available (Gung et al., n.d.). This is useful in industrial inspection because small defects are often difficult to detect and new defect categories may not have enough samples during the early stage of production.
These studies show that few-shot and unknown defect detection can improve model adaptability. However, most methods still require a structured update stage or labelled new samples before the model can learn effectively. This creates a gap for methods that can use test-time information more directly during operation.

## 2.5 Real-World Industrial Anomaly Detection
Many industrial anomaly detection methods are developed and evaluated using benchmark datasets. These datasets are useful for comparing model performance but they may not fully reflect real production conditions. In actual factories, images may be affected by lighting changes, camera position, product variation, surface texture and alignment issues. Defects can also be very small, subtle or difficult to define without comparing the test sample with a normal reference sample.
Recent real-world anomaly detection research shows that the gap between benchmark datasets and actual production environments is a major issue. Real industrial anomalies may require comparison between normal and abnormal samples and they may involve unaligned images, fine-grained defect regions and multiple defect types appearing on the same object (Wang, Zhuo, et al., 2025). This is important because high performance on standard datasets may not always mean strong performance in real factory conditions.
The idea of comparing test samples with normal references is closely related to memory bank based anomaly detection. In this type of method, normal feature patterns are stored and new test samples are compared against these references. If the difference between the test sample and normal reference features is large the sample may be considered abnormal. This approach is useful when labelled defect samples are limited or when defect categories are unknown.
However, real-world anomaly detection still faces practical challenges. First, the stored normal representation may become outdated when production conditions change. Second, fixed thresholds may not work well across different products, lighting conditions or defect types. Third, many studies focus mainly on improving detection accuracy while less attention is given to how the system can update itself during actual operation.

## 2.6 Test-Time Learning and Online Adaptation
Test-time learning provides another direction for improving adaptability. Unlike traditional training, where the model is updated before deployment, test-time learning allows the system to use information available during inference. This is useful in industrial defect detection because new samples appear continuously during operation and fully labelled data may not be immediately available.
In industrial anomaly segmentation, many methods generate anomaly score maps but still require a threshold to convert the score map into a binary defect region. This threshold is often calculated from nominal validation data but it may not remain reliable when defect patterns or production conditions change (Costanzino et al., 2024). This limitation is important for practical inspection because the final result must clearly identify whether a region is defective or normal.
Test-time training has been proposed to improve anomaly segmentation by using information from the test sample itself. Instead of relying only on a fixed threshold, pseudo-labels and extracted features can be used during testing to train a lightweight classifier for the current test sample (Costanzino et al., 2024). This supports the idea that inference-stage data may contain useful information that was not available during the original training stage.
This research follows a related but slightly different direction. Instead of focusing only on refining segmentation masks, this study focuses on an online test-time learning workflow for industrial defect detection. Each incoming test image is compared with stored normal feature patterns. The system checks how different the new image is from the normal references and uses this difference to decide whether the image is normal or abnormal. When a new sample appears to be normal-like, its features can be added into the memory bank to update the normal representation. This allows the system to adapt gradually during operation without full retraining.
This direction is different from many incremental learning studies. Incremental learning usually learns new categories in separate training sessions while test-time learning aims to adapt during inference. For industrial inspection, this is important because the system may need to respond to production changes immediately rather than waiting for a full retraining process.

## 2.7 Low-Code Platform for Practical Implementation
Most studies in industrial defect detection focus on model accuracy, feature learning, forgetting reduction or detection robustness. These are important technical goals but practical implementation is also important. In real industrial settings, the users of an inspection system may include engineers, technicians and quality control staff who may not have strong programming or deep learning experience. If the system requires complex coding, model configuration or manual training procedures, it may be difficult to apply in actual production.
A low-code platform can help reduce this barrier. By providing a more visual and user-friendly workflow, a low-code platform allows users to prepare data, configure the model, run testing and monitor results with less manual coding. This is useful in industrial environments where fast setup, maintainability and ease of use are important.
Although low-code tools are becoming more common in AI applications, they are not widely discussed in the reviewed defect detection studies. Most existing works focus on algorithm-level improvement while fewer studies consider how the proposed method can be made easier for non-specialist users. Therefore, integrating online test-time learning into a low-code platform can help bridge the gap between academic models and practical industrial use.

Based on the reviewed literature, the related studies can be grouped into several main research areas including deep learning-based defect detection, incremental learning, few-shot learning, unknown defect detection, real-world anomaly detection and test-time learning. ## 2.9 Research Gap
Based on the reviewed literature, several research gaps can be identified.
First, many defect detection models are still based on static or closed-set learning. These models may perform well when testing data are similar to training data but they may become less reliable when new defect types, new product batches or new production conditions appear.
Second, incremental learning methods have improved the ability of models to learn new defect categories while reducing catastrophic forgetting. However, most of these methods still depend on staged updates, labelled new data or controlled training sessions. They do not fully support adaptation directly during the inference stage.
Third, few-shot and unknown defect detection methods help address the issue of limited samples. However, many of these methods still require offline learning or additional training steps before the model can adapt effectively.
Fourth, many anomaly detection methods rely on fixed thresholds or predefined normal representations. These approaches may become less accurate when lighting, material surface, camera angle or defect patterns change.
Finally, most reviewed studies focus mainly on technical model performance. Practical issues such as usability, low-code deployment and integration into industrial workflows are less commonly addressed. This creates a gap between research-level models and systems that can be easily used by industrial users.

## 2.10 Summary and Research Direction
The reviewed studies show that deep learning, incremental learning, few-shot learning, unknown defect detection and test-time learning have all contributed to the development of more adaptive defect detection systems. Deep learning improves feature extraction and detection performance. Incremental learning helps models learn new defect categories while reducing catastrophic forgetting. Few-shot and unknown defect detection methods support learning under limited data conditions. Test-time learning further suggests that useful information can be obtained during inference to improve model adaptation.
However, existing approaches still have limitations for practical industrial use. Many methods require offline retraining, staged incremental updates or controlled labelled data. In addition, limited attention has been given to usability and deployment through low-code platforms.
Therefore, this research focuses on developing an online test-time learning method for industrial defect detection using a low-code platform. The proposed approach aims to compare incoming test images with stored normal feature patterns, identify abnormal samples based on feature differences and update the normal memory representation when normal samples are observed. By doing so, the system is expected to adapt gradually during operation without full retraining. At the same time, the low-code platform is intended to make the workflow easier to use for industrial users.
In this sense, the study aims to bridge two gaps: the technical gap between static models and adaptive test-time learning and the practical gap between research algorithms and deployable industrial inspection systems.
